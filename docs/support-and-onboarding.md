# Support And Onboarding

US Video API is being positioned for both self-serve developers and enterprise or agency buyers who need more guided onboarding.

## Typical Onboarding Path

Self-serve teams usually start with:

1. GitHub examples
2. quick API test
3. dashboard and key creation
4. first production-style workflow

Enterprise or agency teams usually need:

1. use-case review
2. integration review
3. workflow validation
4. pricing and scale alignment
5. production onboarding

## What Buyers Usually Need Help With

- choosing the right text-to-video or image-to-video flow
- deciding between direct polling and a webhook-ready internal design
- structuring prompts for ads, ecommerce, or local business use cases
- mapping internal team roles across billing, development, and marketing

## Recommended Internal Roles

- developer owner for API implementation
- marketing or creative owner for prompt and output review
- billing owner for production spend control
- operations owner for workflow and delivery quality

## Practical Buyer Signal

Teams evaluating US Video API should be able to answer:

- can our developers integrate this cleanly
- can our marketing team get ad-ready outputs
- can our business team control billing and team access
- can we audit prompts, jobs, and outputs in our own system

This repository is designed to reduce that first layer of uncertainty.
